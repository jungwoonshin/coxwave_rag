
검토사항 1: 주어진 문제가 LLM 모델의 해결 범위 안에 있는지 검증해야합니다.

    1. 문제의 특성 분석

    언어적 본질: 문제가 언어 이해, 생성, 추론에 중점을 두는지 확인
    필요한 지식 영역: 모델이 학습된 도메인 지식 내에 있는지 평가
    추상화 수준: 개념적, 이론적 이해가 필요한지 vs 구체적 사실 회상이 필요한지

    2. 모델 능력 평가

    모델 크기와 학습 데이터: 더 큰 모델이 더 넓은 범위의 문제를 해결할 가능성이 높음
    컨텍스트 창(Context Window): 문제와 필요한 정보가 모델의 컨텍스트 창에 맞는지 확인
    학습 범위와 한계: 모델의 학습 커트오프 날짜와 학습 데이터 범위 확인
    특화된 능력: 해당 모델이 특정 작업(코딩, 수학 등)에 특화되었는지 확인

    3. 문제 해결 요구사항 분석

    정확성 요구 수준: 100% 정확성이 필요한지 vs 대략적인 답변이 허용되는지
    시간적 요소: 최신 정보가 필요한 문제인지(뉴스, 시장 데이터 등)
    외부 도구 필요성: 웹 검색, 코드 실행, 계산기 등 추가 도구가 필요한지
    감각적 요소: 이미지 분석, 오디오 처리 등 다중 모달 능력이 필요한지

    4. 윤리적/법적 고려사항

    민감한 주제: 문제가 윤리적으로 민감한 영역을 다루는지
    개인정보 처리: 개인 식별 정보나 기밀 데이터 처리가 필요한지
    규제 준수 요구사항: 법적, 규제적 제약이 있는 영역인지(의료, 법률 등)

    5. 벤치마킹 및 테스트

    유사 문제 성능: 유사한 문제에 대한 LLM의 기존 성능 데이터 검토
    샘플 테스트: 소규모 대표 샘플로 모델 성능 시험
    정량적 평가: 정확도, 관련성, 일관성 등을 측정할 수 있는 평가 지표 설정

    6. 한계 및 대안 식별

    실패 가능성: 모델이 해결하지 못할 가능성이 높은 측면 식별
    부분 해결 가능성: 문제의 일부분만 해결 가능한지 평가
    하이브리드 접근법: LLM과 다른 기술을 결합한 접근법 고려

검토사항 2: 데이터셋이 올바른 혹은 잘못된 정보를 포함하고 있는지 검증해야합니다.

    데이터 구조적 측면

    완전성 - 모든 질문에 대응하는 답변이 있는지, 누락된 항목은 없는지 확인
    중복 제거 - 동일하거나 매우 유사한 질문-답변 쌍이 중복되어 있는지 확인
    데이터 형식 일관성 - 모든 레코드가 동일한 형식(필드, 구분자 등)을 따르는지 확인

    콘텐츠 품질 측면

    정확성 - 답변이 실제로 질문에 정확하게 대응하는지 확인
    최신성 - 정보가 최신 상태인지, 오래된 정보가 포함되어 있지 않은지 확인
    완결성 - 답변이 질문을 충분히 해결하는지, 너무 짧거나 불완전하지 않은지 확인
    명확성 - 질문과 답변이 명확하게 표현되었는지 확인

    언어적 측면

    문법 및 맞춤법 - 질문과 답변의 문법, 맞춤법 오류 확인
    일관된 용어 사용 - 동일한 개념이나 용어가 일관되게 사용되는지 확인
    질문 다양성 - 다양한 방식으로 표현된 동일한 의도의 질문이 포함되어 있는지 확인

해결 방안:
=> 파라미터 사이즈가 큰 언어모델을 활용하여, prompt를 통해 위의 기준을 만족하는 데이터셋만 필터링.
